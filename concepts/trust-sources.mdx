Trust sources define which issuers the verifier trusts when validating presented credentials. The EUDI Verifier Endpoint supports multiple trust source types that can be configured per credential type using regex patterns.

## Overview

When a wallet presents credentials, the verifier must validate that the credentials were issued by a trusted authority. This is done by validating the issuer's certificate chain against configured trust sources.

The verifier supports three types of trust sources:

1. **LOTL (List of Trusted Lists)** - A meta-list that references multiple trust lists
2. **Trust List (TL)** - An XML document containing trusted issuer certificates
3. **Keystore** - A local keystore file containing trusted certificates

<Note>
You can combine multiple trust source types for the same pattern. For example, you might use both a LOTL and a keystore to cover different issuer types.
</Note>

## Trust Source Configuration

Trust sources are configured using indexed environment variables starting from `0`:

```bash
# Pattern to match credential types
VERIFIER_TRUSTSOURCES_0_PATTERN="eu.europa.ec.eudi.pid.*|urn:eu.europa.ec.eudi:pid:.*"

# LOTL configuration
VERIFIER_TRUSTSOURCES_0_LOTL_LOCATION="https://example.com/lotl.xml"
VERIFIER_TRUSTSOURCES_0_LOTL_KEYSTORE_PATH="classpath:lotl-key.jks"
VERIFIER_TRUSTSOURCES_0_LOTL_KEYSTORE_TYPE="jks"
VERIFIER_TRUSTSOURCES_0_LOTL_KEYSTORE_PASSWORD="secret"

# Trust List configuration
VERIFIER_TRUSTSOURCES_0_TL_LOCATION="https://example.com/tl.xml"
VERIFIER_TRUSTSOURCES_0_TL_KEYSTORE_PATH="classpath:tl-key.jks"

# Add more trust sources with index 1, 2, etc.
VERIFIER_TRUSTSOURCES_1_PATTERN=".*"
VERIFIER_TRUSTSOURCES_1_LOTL_LOCATION="https://example.com/another-lotl.xml"
```

### Pattern Matching

The `PATTERN` field uses a regex to match against:
- **MSO MDoc credentials**: The `docType` value (e.g., `eu.europa.ec.eudi.pid.1`)
- **SD-JWT VC credentials**: The `vct` value (e.g., `urn:eu.europa.ec.eudi:pid:1`)

Example patterns:
```bash
# Match all PID credentials
"eu.europa.ec.eudi.pid.*|urn:eu.europa.ec.eudi:pid:.*"

# Match all credentials (fallback)
".*"

# Match specific credential type
"eu.europa.ec.eudi.mdl.1"
```

## LOTL (List of Trusted Lists)

A List of Trusted Lists is a meta-list that references multiple trust lists, typically used in federated trust frameworks like the EU Digital Identity Wallet.

### Configuration

```bash
# LOTL location URL
VERIFIER_TRUSTSOURCES_0_LOTL_LOCATION="https://preprod.pki.eudiw.dev/lotl.xml"

# Refresh interval (cron expression, default: every hour)
VERIFIER_TRUSTSOURCES_0_LOTL_REFRESHINTERVAL="0 0 */4 * * *"

# Service type filter (optional)
VERIFIER_TRUSTSOURCES_0_LOTL_SERVICETYPEFILTER="PIDProvider"

# Keystore containing LOTL signing key
VERIFIER_TRUSTSOURCES_0_LOTL_KEYSTORE_PATH="classpath:lotl-key.jks"
VERIFIER_TRUSTSOURCES_0_LOTL_KEYSTORE_TYPE="jks"
VERIFIER_TRUSTSOURCES_0_LOTL_KEYSTORE_PASSWORD="changeit"
```

### Service Type Filters

Filter certificates by provider type:

- `PIDProvider` - Person Identification Data providers
- `QEEAProvider` - Qualified Electronic Attestation of Attributes providers
- `PubEAAProvider` - Public Electronic Attestation of Attributes providers

## Trust List (TL)

A Trust List is an XML document containing X.509 certificates of trusted issuers.

### Configuration

```bash
# Trust List location URL
VERIFIER_TRUSTSOURCES_0_TL_LOCATION="https://preprod.pki.eudiw.dev/tl.xml"

# Refresh interval (cron expression)
VERIFIER_TRUSTSOURCES_0_TL_REFRESHINTERVAL="0 0 */4 * * *"

# Service type filter (optional)
VERIFIER_TRUSTSOURCES_0_TL_SERVICETYPEFILTER="PAAProvider"

VERIFIER_TRUSTSOURCES_0_TL_KEYSTORE_PATH="file:///certs/tl-key.jks"
```

## Keystore

Keystores provide a local source of trusted certificates, useful for testing or private trust frameworks.

### Configuration

```bash
# Keystore path (classpath or file)
VERIFIER_TRUSTSOURCES_0_KEYSTORE_PATH="classpath:trust.jks"

# Keystore type
VERIFIER_TRUSTSOURCES_0_KEYSTORE_TYPE="jks"

# Keystore password (optional)
VERIFIER_TRUSTSOURCES_0_KEYSTORE_PASSWORD="changeit"
```

### Keystore Formats

Supported keystore types:
- `jks` - Java KeyStore format
- `pkcs12` - PKCS#12 format

The trust source provider matches credential types to trust sources:


The implementation matches patterns and returns the appropriate trust configuration:

```kotlin
class TrustSources(
    private val revocationEnabled: Boolean = false,
    private val x5CShouldBeMap: MutableMap<Regex, X5CShouldBe> = mutableMapOf(),
) : ProvideTrustSource {

    override suspend fun invoke(type: String): X5CShouldBe? =
        mutex.withLock {
            x5CShouldBeMap.entries
                .firstOrNull { (pattern, _) -> pattern.matches(type) }
                ?.value
        }
}
```

## Per-Transaction Trust

You can override trust sources for individual transactions by providing an `issuer_chain` during initialization:

```bash
curl -X POST http://localhost:8080/ui/presentations \
  -H "Content-Type: application/json" \
  -d '{
    "dcql_query": { ... },
    "nonce": "unique-nonce",
    "issuer_chain": "-----BEGIN CERTIFICATE-----\nMIIDHTCCAqOg...\n-----END CERTIFICATE-----"
  }'
```

The `issuer_chain` should be a PEM-encoded X.509 certificate chain (including start and end markers).


## Trust Validation Types

The verifier supports two validation types:

```kotlin
enum class ValidationType {
    DS,    // Direct validation using Document Signer certificates
    PKIX,  // PKIX path validation using CA certificates
}
```

*From [VerifierConfig.kt:365-368](/home/daytona/workspace/source/src/main/kotlin/eu/europa/ec/eudi/verifier/endpoint/domain/VerifierConfig.kt#L365-L368)*

- **PKIX** (default): Validates the full certificate chain using CA certificates
- **DS**: Directly trusts Document Signer (DS) certificates without chain validation

## Refresh Intervals

Trust lists are refreshed automatically using cron expressions:

```bash
# Every hour (default)
VERIFIER_TRUSTSOURCES_0_LOTL_REFRESHINTERVAL="0 0 * * * *"

# Every 4 hours
VERIFIER_TRUSTSOURCES_0_LOTL_REFRESHINTERVAL="0 0 */4 * * *"

# Every day at midnight
VERIFIER_TRUSTSOURCES_0_LOTL_REFRESHINTERVAL="0 0 0 * * *"
```

The cron format follows standard 6-field notation: `second minute hour day month weekday`

## Troubleshooting

### Trust Source Not Found

If credentials fail validation with "no trust source found":

1. Check that your pattern matches the credential's docType/vct
2. Verify the trust source URL is accessible
3. Check logs for certificate loading errors

### Certificate Validation Failures

1. Ensure the keystore password is correct
2. Verify the keystore contains the correct signing certificate
3. Check that certificates haven't expired
4. Enable revocation checking if needed
