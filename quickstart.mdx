Get started with the EUDI Verifier Endpoint by building and running the service locally, then test it with your first verification transaction.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Java 17 or later** - Required for building and running the service
- **Docker** (optional) - For containerized deployment
- **Git** - To clone the repository

## Installation

<Steps>
  <Step title="Clone the repository">
    Clone the EUDI Verifier Endpoint repository from GitHub:
    
    ```bash
    git clone https://github.com/eu-digital-identity-wallet/av-srv-web-verifier-endpoint-23220-4-kt.git
    cd av-srv-web-verifier-endpoint-23220-4-kt
    ```
  </Step>

  <Step title="Choose your installation method">
    <CodeGroup>
      ```bash Gradle (Local Development)
      # Build and run the service using Gradle
      ./gradlew bootRun
      ```

      ```bash Docker Build
      # Build a Docker image using Gradle
      ./gradlew bootBuildImage
      ```

      ```bash Docker Compose
      # Run the complete verifier stack (backend + UI + proxy)
      cd docker
      docker-compose up -d
      ```
    </CodeGroup>
    
    <Note>
      The service will start on **port 8080** by default. You can change this by setting the `SERVER_PORT` environment variable.
    </Note>
  </Step>

  <Step title="Verify the service is running">
    Once the service starts, you can verify it's running by accessing:
    
    - **Swagger UI**: http://localhost:8080/swagger-ui
    - **Health Check**: http://localhost:8080/actuator/health
    
    <Tip>
      The Swagger UI provides an interactive interface to explore and test all available API endpoints.
    </Tip>
  </Step>
</Steps>

## Basic Configuration

The verifier can be configured using environment variables. Here are the essential settings to get started:

```bash
# Public URL of the verifier (required for production)
export VERIFIER_PUBLICURL=https://localhost

# Response mode: how authorization responses are expected
export VERIFIER_RESPONSE_MODE=DirectPost

# Client ID configuration
export VERIFIER_ORIGINALCLIENTID=Verifier
export VERIFIER_CLIENTIDPREFIX=pre-registered

# Authorization request signing
export VERIFIER_JAR_SIGNING_KEY=GenerateRandom
export VERIFIER_JAR_SIGNING_ALGORITHM=ES256
```

<Warning>
  In production, both the Verifier API and Wallet API must be exposed over **HTTPS**. The Verifier API should also be protected to allow only authorized access.
</Warning>

For a complete list of configuration options, see the [Configuration Overview](/configuration/overview).

## Your First Transaction

Now that the service is running, let's initialize your first verification transaction to request credentials from a wallet.

<Steps>
  <Step title="Initialize a transaction">
    Send a POST request to initialize a new verification transaction. This example requests an EU Digital Identity PID credential:

    ```bash
    curl -X POST http://localhost:8080/ui/presentations \
      -H "Content-Type: application/json" \
      -H "Accept: application/json" \
      -d '{
        "dcql_query": {
          "credentials": [
            {
              "id": "32f54163-7166-48f1-93d8-ff217bdb0653",
              "format": "mso_mdoc",
              "meta": {
                "doctype_value": "eu.europa.ec.eudi.pid.1"
              },
              "claims": [
                {
                  "path": ["eu.europa.ec.eudi.pid.1", "family_name"]
                }
              ]
            }
          ],
          "credential_sets": [
            {
              "options": [
                ["32f54163-7166-48f1-93d8-ff217bdb0653"]
              ],
              "purpose": "We need to verify your identity"
            }
          ]
        },
        "nonce": "test-nonce-123",
        "jar_mode": "by_reference",
        "request_uri_method": "post"
      }'
    ```

    <Note>
      This request uses **DCQL (Digital Credentials Query Language)** to specify which credentials and claims are required.
    </Note>
  </Step>

  <Step title="Get the transaction response">
    The API returns transaction details including the authorization request:

    ```json
    {
      "transaction_id": "STMMbidoCQTtyk9id5IcoL8CqdC8rxgks5FF8cqqUrHvw0IL3AaIHGnwxvrvcEyUJ6uUPNdoBQDa7yCqpjtKaw",
      "client_id": "pre-registered:Verifier",
      "request_uri": "https://localhost:8080/wallet/request.jwt/5N6E7VZsmwXOGLz1Xlfi96MoyZVC3FZxwdAuJ26DnGcan-vYs-VAKErioQ58BWEsKlVw2_X49jpZHyp0Mk9nKw",
      "request_uri_method": "post"
    }
    ```

    <Tip>
      Save the `transaction_id` - you'll need it to retrieve the wallet's response later.
    </Tip>
  </Step>

  <Step title="Generate a QR code (optional)">
    For cross-device flows, you can request a QR code instead of JSON by changing the `Accept` header:

    ```bash
    curl -X POST http://localhost:8080/ui/presentations \
      -H "Content-Type: application/json" \
      -H "Accept: image/png" \
      -d '{
        "dcql_query": { ... },
        "nonce": "test-nonce-123",
        "jar_mode": "by_reference"
      }' \
      --output qrcode.png
    ```

    This generates a PNG image containing the authorization request that can be scanned by a wallet application.
  </Step>

  <Step title="Retrieve the wallet response">
    After the wallet submits its response, retrieve it using the transaction ID:

    ```bash
    curl http://localhost:8080/ui/presentations/{transaction_id}?response_code={response_code}
    ```

    <Note>
      The `response_code` parameter is only required for same-device flows. For cross-device flows, you can poll this endpoint until a response is available.
    </Note>
  </Step>
</Steps>

## Docker Compose Deployment

For a complete verifier setup including the UI and SSL termination, use Docker Compose:

```yaml docker/docker-compose.yaml
version: '3.3'

services:
  verifier:
    image: ghcr.io/eu-digital-identity-wallet/av-srv-web-verifier-endpoint-23220-4-kt:latest
    container_name: verifier-backend
    ports:
      - "8080:8080"
    environment:
      VERIFIER_PUBLICURL: "https://localhost"
      VERIFIER_RESPONSE_MODE: "DirectPost"
      VERIFIER_JAR_SIGNING_KEY: "GenerateRandom"

  verifier-ui:
    image: ghcr.io/eu-digital-identity-wallet/eudi-web-verifier:latest
    container_name: verifier-ui
    ports:
      - "4300:4300"
    environment:
      - HOST_API="http://verifier:8080"

  haproxy:
    image: haproxy:2.7.2
    container_name: haproxy
    ports:
      - "443:443"
    volumes:
      - ./haproxy.conf:/usr/local/etc/haproxy/haproxy.cfg
      - ./haproxy.pem:/etc/ssl/certs/mysite.pem
```

Start the complete stack:

```bash
cd docker
docker-compose up -d
```

Stop the stack:

```bash
cd docker
docker-compose down
```
