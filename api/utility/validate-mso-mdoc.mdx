## Overview

This utility endpoint validates an MSO MDoc DeviceResponse to ensure it meets security and integrity requirements. Use this endpoint to verify credentials before accepting them in your application.

<Info>
This is a utility endpoint for testing and validation purposes. For production credential verification, use the main [Verifier API](/api/verifier/initialize-transaction) flow.
</Info>

## Validations performed

The endpoint performs the following security checks:

<Steps>
  <Step title="Format validation">
    Verifies the provided value is an MSO MDoc DeviceResponse encoded using Base64 URL Safe encoding
  </Step>
  <Step title="Validity period">
    Checks that MSO MDoc Documents contain non-expired ValidityInfo
  </Step>
  <Step title="Document type">
    Verifies documents are of the expected docType
  </Step>
  <Step title="Digest verification">
    Validates that IssuerSigned items contain valid Digests
  </Step>
  <Step title="Issuer trust">
    Confirms documents are signed by a trusted Issuer (checked against a configured X5C chain)
  </Step>
</Steps>

## Request

<ParamField body="device_response" type="string" required>
  The MSO MDoc DeviceResponse encoded using Base64 URL Safe encoding
</ParamField>

<ParamField body="issuer_chain" type="string">
  Optional X5C certificate chain for issuer verification
</ParamField>

## Response

<Tabs>
  <Tab title="Success">
    When validation succeeds, the endpoint returns a simplified view of the successfully validated MSO MDoc Documents.

    <ResponseField name="[].docType" type="string" required>
      The document type identifier (e.g., `eu.europa.ec.eudi.pid.1`)
    </ResponseField>

    <ResponseField name="[].attributes" type="object" required>
      Object containing namespaced attributes from the document
      
      <Expandable title="properties">
        <ResponseField name="[namespace]" type="object">
          Key-value pairs of attributes within the namespace. Each attribute represents a verified claim from the credential.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseExample>
    ```json Success response
    [
      {
        "docType": "eu.europa.ec.eudi.pid.1",
        "attributes": {
          "eu.europa.ec.eudi.pid.1": {
            "age_over_18": true
          }
        }
      }
    ]
    ```
    </ResponseExample>
  </Tab>

  <Tab title="Error">
    When validation fails, the endpoint returns a detailed error response identifying all violations found.

    <ResponseField name="type" type="string" required>
      Error type identifier (e.g., `InvalidDocuments`)
    </ResponseField>

    <ResponseField name="invalidDocuments" type="array">
      List of documents that failed validation
      
      <Expandable title="properties">
        <ResponseField name="index" type="number">
          Zero-based index of the invalid document in the DeviceResponse
        </ResponseField>
        
        <ResponseField name="documentType" type="string">
          The document type that failed validation
        </ResponseField>
        
        <ResponseField name="errors" type="string[]">
          Array of error codes describing what validations failed (e.g., `InvalidIssuerSignedItems`, `ExpiredDocument`)
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseExample>
    ```json Error response
    {
      "type": "InvalidDocuments",
      "invalidDocuments": [
        {
          "index": 0,
          "documentType": "eu.europa.ec.eudi.pid.1",
          "errors": [
            "InvalidIssuerSignedItems"
          ]
        }
      ]
    }
    ```
    </ResponseExample>
  </Tab>
</Tabs>

## Common errors

<AccordionGroup>
  <Accordion title="InvalidDocuments">
    One or more documents in the DeviceResponse failed validation. Check the `invalidDocuments` array in the error response for specific error codes:
    
    - `ExpiredDocument` — The document's ValidityInfo indicates it has expired
    - `InvalidIssuerSignedItems` — The digest verification failed for one or more signed items
    - `UnexpectedDocType` — The document type doesn't match expected types
    - `UntrustedIssuer` — The issuer's certificate is not in the trusted chain
  </Accordion>
  
  <Accordion title="InvalidFormat">
    The provided `device_response` parameter is not valid Base64 URL Safe encoded data or does not represent a valid MSO MDoc DeviceResponse structure.
  </Accordion>
  
  <Accordion title="MissingParameter">
    The required `device_response` parameter was not provided in the request body.
  </Accordion>
</AccordionGroup>
