## Overview

The EUDI Verifier Endpoint is a backend RESTful service that acts as a Verifier/Relying Party (RP) trusted endpoint implementing the [OpenID4VP (1.0)](https://openid.net/specs/openid-4-verifiable-presentations-1_0-final.html) protocol. It enables secure verification of digital credentials presented by EU Digital Identity Wallets.

This service works alongside the [EUDI Web Verifier UI](https://github.com/eu-digital-identity-wallet/eudi-web-verifier) to provide a complete verification solution for both same-device and cross-device presentation flows.

<Note>
This is part of the [EUDI Wallet Reference Implementation](https://github.com/eu-digital-identity-wallet/.github/blob/main/profile/reference-implementation.md) project.
</Note>

## Architecture

The verifier endpoint exposes two primary APIs:

- **Verifier API** — Used by verifier applications to initiate transactions and retrieve wallet responses
- **Wallet API** — Used by wallet applications to retrieve authorization requests and post responses

Both APIs must be exposed over HTTPS in production environments.

## Use Cases

The EUDI Verifier Endpoint supports various credential verification scenarios:

- **Identity Verification** — Verify Personal Identification Data (PID) from EU Digital Identity Wallets
- **Age Verification** — Request and verify age-related attributes without revealing full identity
- **Qualified Attestations** — Verify Qualified Electronic Attestations of Attributes (QEAA)
- **Custom Credentials** — Request any credential type supported by the wallet

## Standards & Specifications

This implementation follows these key specifications:

- [OpenID for Verifiable Presentations 1.0](https://openid.net/specs/openid-4-verifiable-presentations-1_0-final.html)
- [JWT-Secured Authorization Request (JAR)](https://www.rfc-editor.org/rfc/rfc9101.html)
- [ISO/IEC 18013-5:2021 Mobile Driving License (mDL)](https://www.iso.org/standard/69084.html)
- [SD-JWT-based Verifiable Credentials (SD-JWT VC)](https://datatracker.ietf.org/doc/draft-ietf-oauth-sd-jwt-vc/)
